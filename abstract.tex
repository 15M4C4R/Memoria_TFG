\chapter*{Abstract}
During the first decade of the 21st century, browser games represented the main gateway to digital leisure for a generation of young people, democratizing entertainment thanks to their immediacy and accessibility. Platforms driven by technologies such as \textit{Adobe Flash} allowed millions of users to enjoy simple and direct multiplayer experiences, without entry barriers or expensive hardware. However, consumption habits progressively changed due to the industry's shift towards large-scale productions and closed mobile ecosystems, culminating in the definitive end of support for Flash Player in 2020 \cite{adobe_flash_eol}. The present Bachelor's Thesis, \textit{King and Peasant}, arises from the personal motivation to recover that nostalgic essence, reconstructing the social browser game experience under the standards of modern software engineering. The objective is not to compete with the commercial industry, but to create a technological tribute that gives back to the user the ability to "jump in and play" with friends instantly.

To achieve this goal, the asymmetric strategic card game \textit{King and Peasant} has been adapted to the web environment through an agile development cycle based on Scrum. This implementation has been carried out using modern technologies such as React, Express, Node.js, MariaDB, and WebSockets, under the Model-View-Controller (MVC) architectural pattern. As a result, a functional and dockerized web application has been deployed, integrating a user management system, multiplayer waiting rooms (lobbies), and a real-time rules engine, demonstrating that it is possible to use current web technologies to offer an interactive and fluid experience, supported by a solid and scalable technical foundation that meets the standards of professional software engineering.
